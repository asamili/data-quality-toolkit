## With coverage (recommended; uses `addopts`)

First install test/dev deps so `pytest-cov` is available:

```powershell
pip install -e .[test]      # or: pip install -e .[dev]
```

Run everything:

```powershell
pytest -vv -x --maxfail=1 -rA --tb=long
```

Just unit tests:

```powershell
pytest tests/unit -vv -x --maxfail=1 -rA --tb=long
```

Just integration tests:

```powershell
pytest tests/integration -vv -x --maxfail=1 -rA --tb=long
```

> Note: **don’t** use `-p no:cov` anymore—`addopts` contains `--cov` flags, and disabling the plugin makes those flags invalid.

## Without coverage (one-off)

Override `addopts` for this run (keeps your strict flags, drops coverage):

```powershell
pytest -o addopts="-ra --strict-markers --strict-config" tests/unit -vv -x --maxfail=1 -rA --tb=long
pytest -o addopts="-ra --strict-markers --strict-config" tests/integration -vv -x --maxfail=1 -rA --tb=long
```

(or the blunt hammer:)

```powershell
pytest -o addopts="" tests/unit -vv -x --maxfail=1 -rA --tb=long
```

## Optional convenience

Set for the session:

```powershell
$env:PYTEST_ADDOPTS="-ra --strict-markers --strict-config"
pytest tests/integration -vv -x --maxfail=1 -rA --tb=long
```
